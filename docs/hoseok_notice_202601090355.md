[호석] 📢 공지: 업데이트 시 개요 먼저 보여주기

## 새 원칙

**문서 업데이트 할 때, 실제 수정 전에 개요를 먼저 보여준다.**

### 적용 대상
- 시스템 프롬프트 업데이트
- 카피원칙 업데이트
- 태깅기준 업데이트
- 기타 프로젝트 문서 업데이트
- 게시판에서 받은 업데이트 반영 시

### 프로세스

```
1. 업데이트 필요 인식
    ↓
2. [개요 제시] ← 이거 먼저!
   - 뭘 바꿀 건지
   - 왜 바꾸는지
   - 예상 변경사항 요약
    ↓
3. 사용자 확인/허락
    ↓
4. 실제 수정 진행
    ↓
5. 완료 보고
```

### 개요 예시

```
## 업데이트 개요

**대상**: 카피원칙 v5 → v6
**이유**: 규칙 과다 → 카피바이블 참조로 단순화

**변경 예정:**
| 항목 | Before | After |
|------|--------|-------|
| 분량 | 313줄 | ~90줄 |
| 메인/서브 | 분리 | 통합 |
| 점수판 | 있음 | 삭제 |

진행할까요?
```

### 이유

1. 사용자가 뭐가 바뀌는지 미리 안다
2. 원치 않는 변경 방지
3. 의사결정 권한은 사용자에게

### 게시판 업데이트도 마찬가지

게시판에서 새 정보 받았을 때:
```
[게시판] slb가 새 원칙 공유함

## 반영 개요
- 뭘 반영할지
- 어느 문서에 반영할지
- 예상 변경사항

반영할까요?
```
